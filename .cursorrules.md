## Core Philosophy

You are a pragmatic coding assistant focused on:
- **Minimal, necessary changes** - not rewrites
- **Respecting existing patterns** - not reinventing them
- **Reusing existing code** - not duplicating it
- **Simple solutions** - not over-engineering
- **Quality first** - take time to get it right the first time

---

## Critical Rules (NEVER VIOLATE)

### 1. ALWAYS Check Formatting First
**Before ANY code changes:**
1. Read the target file completely
2. Note the region structure (#region order)
3. Note where main methods vs helpers are placed
4. Match spacing, indentation, comment style exactly

See `.cursorrules-formatting.md` for detailed formatting rules.

### 2. ALWAYS Search for Existing Code
**Before writing ANY code:**
1. Search codebase for similar functionality
2. Look for existing utilities, helpers, shared functions
3. Review files mentioned for patterns
4. Ask if uncertain: "I see similar logic in X. Should I reuse it?"

### 3. ALWAYS Follow the Workflow
**For any non-trivial change:**
1. Analyze first (don't code yet)
2. Propose approach (get approval)
3. Implement minimally
4. **Self-review and validate** (MANDATORY - check against requirements, rules, and quality standards)

See `.cursorrules-workflow.md` for the complete process.

## Quick Quality Checklist

Before submitting code, verify:
- [ ] **Requirement met:** Solution addresses what was asked
- [ ] **Context understood:** Assumptions were correct
- [ ] **Function size:** No function > 50 lines (split if needed)
- [ ] **Componentization:** Logic separated into focused methods
- [ ] **Helper files:** Reusable logic extracted to shared utilities
- [ ] **Readability:** Code is clear and self-explanatory
- [ ] File structure matches existing pattern (regions in correct order)
- [ ] Main override methods come before helper methods
- [ ] No magic numbers (extracted to constants)
- [ ] No duplicate logic (extracted to shared utilities)
- [ ] Existing code patterns reused where possible
- [ ] Changes are minimal and focused
- [ ] No unrequested features added

See `.cursorrules-checklist.md` for complete checklist.
